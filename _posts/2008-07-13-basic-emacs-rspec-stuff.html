I’ve been using Emacs almost exclusively for the past couple of

months. I used to be a TextMate fanatic, but then I started working

at a place that uses IntelliJ and I was reminded of just how

productive split panes are. TM doesn’t have it, and I’m not about to

use IntelliJ at home, so I decided to give Emacs a shot (yes I know

vim can do it too. “You people” can go away :P)

Anyway, I came for the split panes and stuck around for the ultimate

programmability. In my experimentation with Emacs and Lisp, I’ve

found a little bit of time to make Emacs better for my

Ruby/RSpec development. Nothing special here, just commands to run

run a whole spec file or an individual spec. The output shows up in

emacs, and when a test fails, the lines are clickable so you can jump

right to the test or error. This is all in my basic RSpec

mode.

In addition, I’ve created some

snippets

using

yasnippet.

Right now they’re just for before(:each), describe “some string”,

describe SomeClass, and it “should do something”. I’ve never

gotten into snippets for things like should_receive and stuff.

You can do all of this stuff with other editors, of course. I’m just

sharing in case anyone has started to use or is interested in using

Emacs, but misses some of the little TextMate niceties.
